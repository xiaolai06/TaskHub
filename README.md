# 智汇轻营 TaskHub

面向一人公司 / 自由职业者的 **AI 驱动智能经营平台**。

不是又一个项目管理工具——而是你的 AI 经营合伙人。

## ✨ 核心亮点

- **🤖 AI Agent** — 80+ 工具（16 个功能组）+ 智能关键词路由 + 意图识别提示词 + 用户记忆系统，AI 可直接操作全部业务数据
- **📅 智能排期** — 甘特图 + 插单模拟 + 冲突检测 + 延期预警 + 可配置每日工时上限
- **💰 全链路财务** — 流水账本 → 回款管理 → 订阅追踪 → 应收账款 → 账龄分析 → 同比环比趋势
- **📊 四维经营看板** — 财务概览 / 项目分析 / 任务效率 / 客户洞察，多图表可视化
- **📨 多渠道推送** — 站内通知 + SMTP 邮件 + 企业微信 / 飞书 / 钉钉 / Slack Webhook
- **⏰ 8 个定时任务** — 晨间简报、客户雷达、利润脉搏、成本预警、到期提醒、周报、健康检查、记忆沉淀
- **🔍 行业洞察** — 多源聚合搜索（GitHub / Hacker News / Dev.to / DuckDuckGo / SearXNG）+ AI 分析简报
- **🎙️ 语音输入** — 多供应商 STT 转写（Groq / OpenAI / SiliconFlow / Ollama）
- **🔒 企业级安全** — AES-256-GCM 加密、JWT 静默续签、bcrypt 加盐哈希、数据隔离、Zod 全量校验、限频保护

## 🛠 技术栈

| 层 | 技术 |
|---|------|
| 前端 | Next.js 16 + React 19 + Tailwind CSS 4 + shadcn/ui + React Query + Zustand |
| 后端 | Express 5 + Prisma 6 + SQLite + Zod 3 |
| AI | OpenAI SDK（兼容 DeepSeek / OpenAI / Claude / Ollama 等 23+ 供应商，支持 SSE 流式 + 工具调用 + 文件上传） |
| 推送 | nodemailer (SMTP) + 企业微信 / 飞书 / 钉钉 / Slack Webhook |
| 搜索 | SearXNG（自建） + Tavily + SerpAPI + DuckDuckGo + 搜狗 |
| 定时 | node-cron |
| 安全 | JWT (httpOnly Cookie) + bcryptjs + AES-256 + IP 限频 |

## 🚀 快速开始

### 环境要求

- Node.js 18+（推荐 20 LTS）
- npm

### 1. 后端

```bash
cd backend
npm install
cp .env.example .env    # 编辑 .env 填写必要配置
npx prisma db push
npx prisma generate
npm run dev             # http://localhost:3001
```

`.env` 最少需要：

```env
DATABASE_URL="file:./dev.db"
JWT_SECRET="your-random-secret-at-least-32-chars"
ENCRYPTION_KEY="your-32-char-aes-key-here"
FRONTEND_URL="http://localhost:3002"
```

### 2. 前端

```bash
cd frontend
npm install
npm run dev             # http://localhost:3002
```

前端通过 Next.js Rewrite 自动代理 `/api/*` 到后端。

### 3. 初始化测试数据（可选）

```bash
cd backend
npx prisma db seed      # 填充 2 用户 / 2 项目 / 6 任务 / 5 成本 / 3 客户
```

### 智能机制

| 机制 | 说明 |
|------|------|
| **关键词路由** | 根据用户消息中的关键词，按需加载对应工具组，减少 Token 消耗 |
| **意图识别** | Prompt Selector 自动识别消息意图（创建/分析/排期/搜索/通用），路由到最优系统提示词 |
| **用户记忆** | 自动从对话中提取用户偏好和知识，带置信度评分，下次对话智能注入（最多 8 条） |
| **模型降级** | 默认模型失败时自动切换到强力模型，保障可用性 |
| **Token 管理** | 消息裁剪到 8000 Token 预算，保留工具调用对的完整性 |
| **写操作确认** | 工具执行写操作前需用户确认（可配置自动/手动） |

## 🔒 安全架构

| 安全层 | 实现方式 |
|--------|----------|
| **认证** | JWT httpOnly Cookie（SameSite=Lax） + 7 天有效期 + 静默续期（< 1 天自动刷新） |
| **密码** | bcryptjs 10 轮加盐哈希 |
| **密钥存储** | AES-256-GCM 加密，不存明文 |
| **输入校验** | 所有写入端点 Zod Schema 校验，结构化错误返回 |
| **数据隔离** | 所有数据查询强制 `ownerId` 过滤，不可跨用户访问 |
| **限频保护** | IP 滑动窗口：登录 5 次/分钟，API 60 次/分钟 |
| **CORS** | 严格限制为 `FRONTEND_URL` |
| **Webhook** | `X-Webhook-Secret` 头部验证 |
| **错误脱敏** | API Key 等敏感信息自动从错误消息中剥离 |
