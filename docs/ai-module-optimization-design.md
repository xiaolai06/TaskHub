# TaskFlow+ AI 模块优化设计方案（最终版）

> 核心定位：双栏布局优化，左侧升级为智能工作台（保留 Tab），右侧对话体验升级
> 目标用户：一人公司

---

## 一、改动总览

| 区域 | 当前问题 | 优化方向 |
|------|----------|----------|
| 左侧 Tab | 概览/任务/快捷，Tab 内容价值低 | 概览/客户/历史，三个 Tab 各有价值 |
| 左侧概览 Tab | 纯项目列表罗列 | 智能摘要卡片 + 置顶快捷提问 |
| 左侧任务 Tab | 和任务页数据重复 | **替换为客户 Tab**，展示客户列表+跟进状态 |
| 左侧快捷 Tab | 只有 6 个按钮 | **取消**，快捷提问分散到概览 Tab + 对话区空状态 + 输入框上方 |
| 会话历史 | 全屏浮层遮挡对话 | 移入左侧第三个 Tab「历史」，加搜索和智能命名 |
| 新建对话 | 顶部小 + 按钮 | 历史 Tab 底部醒目按钮 + 顶部快捷入口 |
| 模型切换 | 要去设置页 | 左侧底部固定显示，下拉选择，数据来自 API |
| 用户头像 | Bot/User 通用图标 | 读取 useAuth 中的用户头像/名称 |
| 右侧对话气泡 | 灰底/紫底单调 | 白底边框 AI 气泡 + 用户头像 + 时间戳 |
| 工具调用 | 每步一行啰嗦 | 折叠为一行摘要 |
| 空状态 | 一个 Bot 图标 | 大卡片引导 + 输入框 |
| 快捷提问 | 6个固定不变 | 按时间动态变化 + 分场景散布 |

## 三、模型切换器（左侧底部）

### 数据来源

模型列表 = 用户在设置页配置的模型。来源有两个：

1. **静态回退**：`GET /api/settings/ai/models?provider=deepseek`
   → 返回 `{ models: [{id, name, tier}], baseUrl }`

2. **动态获取**：用户在设置页点了「获取模型列表」后，后端调用供应商 API
   → 存入设置，前端从同样的接口读

前端不需要区分来源，统一从 `GET /api/settings/ai/models` 拿。

### UI 设计

```
┌──────────────────────────────┐
│  ────────────────────────    │  ← 分隔线
│                              │
│  🤖 DeepSeek V3        ▼    │  ← 当前模型，始终可见
│     deepseek-chat            │  ← 模型 ID，小字灰色
└──────────────────────────────┘
```

点击展开下拉菜单：

```
┌──────────────────────────────┐
│  ⚡ 快速模型（响应快）        │
│  ✓ DeepSeek V3              │  ← 当前选中，打勾
│    deepseek-chat             │
│    GPT-4o Mini              │
│    gpt-4o-mini              │
│                              │
│  🧠 强力模型（分析复杂任务）   │
│    DeepSeek R1               │
│    deepseek-reasoner         │
│    Claude 4 Sonnet           │
│    claude-sonnet-4-6        │
│                              │
│  ────────────────────────    │
│  ⚙️ 配置更多模型 →           │  ← 跳转设置页
└──────────────────────────────┘
```

### 分组逻辑

模型从 API 返回时带 `tier` 字段：
- `tier: 'fast'` → 放入「⚡ 快速模型」
- `tier: 'powerful'` → 放入「🧠 强力模型」
- `tier: 'balanced'` → 放入「⚡ 快速模型」（平衡型偏快）

### 切换行为

```typescript
// useAiChat 中维护 selectedModel
const [selectedModel, setSelectedModel] = useState<string | undefined>();

// sendMessage 时传给后端
fetch('/api/llm/chat/stream', {
  body: JSON.stringify({ message, sessionId, model: selectedModel }),
});

// 不传 model → 后端用 default_model（设置页配置的默认模型）
// 传了 model → 后端用指定模型
```

选中强力模型时，输入框左下角显示小标记：

```
🧠 使用 DeepSeek R1（回答可能较慢但更详细）
[输入你的问题___________________] [→]
```

### 位置

固定在左侧面板底部，不管切换到哪个 Tab 都始终可见。分隔线以上是 Tab 内容区，分隔线以下是模型切换器。

## 四、用户头像展示

### 数据来源

```typescript
// useAuth() 返回的 user 对象
interface User {
  id: string;
  email: string;
  name: string;
  role: string;
  avatar?: string | null;  // 头像 URL，可能为 null
}
```

`GET /api/auth/me` 返回 `{ user: { id, email, name, role, avatar } }`。

### 头像渲染逻辑

```typescript
function getUserAvatar(user: User | null): ReactNode {
  if (!user) return <UserIcon />;  // 未登录，显示通用图标
  
  if (user.avatar) {
    // 有头像图片 → 显示图片
    return <img src={user.avatar} alt={user.name} className="rounded-full" />;
  }
  
  // 没有头像 → 显示首字母头像
  const initial = user.name?.charAt(0)?.toUpperCase() || 'U';
  return (
    <div className="flex items-center justify-center rounded-full bg-indigo-100 text-indigo-600 font-medium">
      {initial}
    </div>
  );
}
```

### 应用位置

**对话区消息气泡：**

```
当前：
  🤖 [AI 消息]               [用户消息] 👤
  （通用 Bot 图标）          （通用 User 图标）

优化后：
  🤖 [AI 消息]               [用户消息] [H]
  （AI 保持 Bot 图标）       （用户真实头像/首字母）
```

用户消息右侧显示真实头像：
- 有头像图片 → 28px 圆形头像
- 无头像 → 28px 圆形首字母（如「H」= 何恺）

**左侧面板底部（可选）：**

在模型切换器上方可以显示当前用户的小信息条：

```
┌──────────────────────────────┐
│  [H]  何恺                   │  ← 头像 + 名字
│       管理员                 │
└──────────────────────────────┘
```

但这个是可选的，不是核心需求。

```
┌──────────────────────────────────────┐
```

```
其他时间:       [🌅 今日简报] [📊 项目进度] [👤 客户分析]
```

- **📋 复制** → 复制整条消息的纯文本，成功时 toast 提示「已复制」
- **↻ 重新生成** → 删除当前 AI 消息，用上一条用户消息重新发送

## 十一、数据流

```
面板打开
  ├→ AiSidebar render
  │   ├→ SmartDigest: fetch tasks + customers + finance (并行)
  │   ├→ CustomerTab: fetch customers (懒加载，切到 Tab 时请求)
  │   ├→ HistoryTab: fetch sessions (面板打开时请求)
  │   └→ ModelSwitcher: fetch AI settings models (面板打开时请求)
  │
  └→ ChatArea render
      ├→ 有当前会话 → 加载历史消息
      └→ 无当前会话 → 显示空状态

用户发送消息
  ├→ useAiChat.sendMessage(message, sessionId, model?)
  ├→ SSE stream → 更新消息列表
  ├→ 工具调用折叠到 ToolCallBar
  ├→ 写操作完成 → 刷新相关缓存
  └→ 刷新会话列表（左侧历史 Tab）

切换模型
  ├→ ModelSwitcher onChange
  ├→ 更新 useAiChat selectedModel
  └→ 下次 sendMessage 带上新模型
```

## 十二、总结

| 优化区域 | 核心改动 |
|----------|----------|
| **左侧 Tab** | 概览（智能摘要+快捷提问+项目） / 客户（跟进提醒+客户列表） / 历史（会话列表+搜索+新建） |
| **模型切换** | 左侧底部，数据来自设置 API 的模型列表，分快速/强力两组 |
| **用户头像** | 从 useAuth 读取 user.avatar/name，图片优先，无图片显示首字母 |
| **快捷提问** | 分散到 3 处：概览 Tab 3 个 + 空状态 3 个引导卡片 + 输入框上方动态 chips |
| **新建对话** | 历史 Tab 顶部醒目按钮 + 顶部栏 + 图标按钮 |
| **会话命名** | 自动用第一条用户消息截断 20 字命名 |
| **右侧气泡** | AI 白底边框阴影 / 用户紫底白字 + 用户真实头像 + 时间戳 |
| **工具调用** | 折叠为一行摘要，可展开 |
| **面板宽度** | 保持 900px（左 320 + 右 580），或微调到 860px |
