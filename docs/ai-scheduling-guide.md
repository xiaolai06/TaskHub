# 智能排期 — 功能说明与 AI 排期设计

## 一、什么是智能排期

智能排期是排期视图中的 AI 辅助功能，核心思路：

> **算法告诉你"现状是什么"，AI 告诉你"应该怎么办"。**

排期算法只能基于已有数据做数学计算（优先级排序 + 工时分配），但 AI 能：
- 理解项目描述，判断任务拆分是否合理
- 结合业务背景给出优先级建议
- 用自然语言解释排期问题和解决方案

---

## 二、当前实现

### 入口

排期视图工具栏有一个「智能安排」按钮（紫色，带 ✨ 图标）。

### 触发逻辑

点击后，前端把以下信息发送给 AI：

```
请帮我分析项目「XXX」的排期：
当前每日工时上限 8h，8 个任务共 32h，延期 2 个，冲突 1 天。

项目说明：这是一个机器视觉模型开发项目，目标是...  ← 如果有项目描述

请结合项目说明，分析任务拆分是否合理、工时估算是否准确、有没有遗漏的任务。

请给出具体的排期优化建议...
```

### AI 收到后调用的工具

| 工具 | 回答什么 | 触发场景 |
|------|---------|---------|
| `get_schedule_advice` | 排期建议 + 优先级排序 | "先做哪个？""能按时完成吗？" |
| `suggest_rebalance` | 冲突解决 + 延期处理 | "排期有什么问题？" |
| `evaluate_insertion` | 插单影响评估 | "加一个新任务会影响谁？" |

### AI 侧边栏快捷问题

在 AI 助手的「排期」标签页中，有 5 个预设问题：

| 按钮 | 发送给 AI 的内容 |
|------|-----------------|
| 延期任务 | "有什么延期的任务？" |
| 今日焦点 | "今天的工作焦点是什么？先做哪个？" |
| 排期健康度 | "帮我检查一下排期健康度，有没有延期或冲突？" |
| 插单评估 | "我想接一个新项目，会影响现有排期吗？" |
| 什么时候有空 | "我什么时候有空？最近一周的排期情况怎么样？" |

## 三、禁用逻辑

「智能安排」按钮在以下情况被禁用（灰色不可点击）：

| 条件 | 原因 |
|------|------|
| 没有选择项目（全部项目模式下 effectiveProjectId 为空） | AI 需要知道分析哪个项目 |
| AI 正在加载中 | 防止重复请求 |

### 无排期数据时的行为

当项目里没有填写实际工时的任务时，排期视图显示空状态。此时点击「智能安排」，AI 会收到提示：

```
当前没有已记录实际工时的任务，无法自动排期。
请根据项目说明，帮我拆解任务并给出排期建议。
```

AI 会转而基于**项目描述**来分析和建议，而不是基于排期数据。

## 四、AI 基于项目描述做排期（设计思路）

### 问题

当前排期依赖任务数据（actualHours），但很多项目刚建立时：
- 只有项目描述，还没有拆分任务
- 任务拆了但没填实际工时
- 不知道该先做什么、每件事要多久

### 解决方案：AI 读取项目描述 → 生成排期建议

```
项目描述: "开发一个机器视觉模型，需要数据采集、模型训练、
          部署上线三个阶段，预计 3 个月完成"
          │
          ↓ AI 分析
          │
AI 建议:
  1. 数据采集 (2 周, HIGH)
     - 数据爬取脚本    8h
     - 数据清洗标注    16h
     - 数据质量检查    4h
  2. 模型训练 (3 周, URGENT)
     - 模型选型调研    8h
     - 训练脚本开发    16h
     - 模型训练调参    40h
     - 模型评估        8h
  3. 部署上线 (1 周, HIGH)
     - API 接口开发    8h
     - 部署配置        4h
     - 压力测试        4h
  
  总工时: ~116h
  建议每日 8h，约 15 个工作日完成
  建议先做数据采集，因为模型训练依赖数据质量。
```

### 实现方式

前端发送的 AI 消息中已经包含了项目描述：

```typescript
if (hasProjectDesc) {
  parts.push(`\n项目说明：${selectedProject.description}`);
  parts.push('\n请结合项目说明，分析任务拆分是否合理、工时估算是否准确、有没有遗漏的任务。');
}
```

AI 助手收到后会：
1. 读取项目描述，理解项目目标和范围
2. 检查现有任务是否覆盖了项目需求
3. 如果任务不完整，建议补充遗漏的任务
4. 如果没有任务，建议完整的任务拆分方案
5. 给出排期建议（优先级、工时、先后顺序）

## 五、提示词文件说明

系统中有 3 个排期相关的提示词文件：

| 文件 | 用途 | 被谁调用 |
|------|------|---------|
| `system-schedule.txt` | 排期专用系统提示词 | AI 对话触发排期关键词时 |
| `schedule-suggest.txt` | 排期建议专用提示词 | 生成排期优化建议时 |
| `task-parse.txt` | 任务解析提示词 | 用户用自然语言描述任务时 |

### system-schedule.txt

定义 AI 的排期助手角色，包含：
- 能做什么（查排期、今日焦点、延期检测、插单模拟）
- 排期规则（优先级排序、每日工时上限）
- 工作方式（先理解问题 → 调工具 → 用时间线解释）
- 相关工具列表（9 个排期工具）
- 回复风格（按时间组织、标注延期天数、给执行建议）

### schedule-suggest.txt

排期优化建议的专用提示词，指导 AI：
- 从 4 个维度分析（延期风险、工时分配、优先级、瓶颈）
- 5 种建议类型（调优先级、拆任务、调工时、调资源、缩减范围）
- 输出格式（状态总结 → 风险点 → 建议 → 预期效果）

### task-parse.txt

自然语言转结构化任务的提示词，指导 AI：
- 提取 title、priority、estimatedHours、dueDate、projectId
- 多任务拆分规则
- 工时估算标准（简单 1-2h / 中等 4-8h / 复杂 16-40h）

## 六、完整数据流

```
用户点击「智能安排」
  │
  ├─ 有排期数据 → 发送排期统计 + 项目描述 → AI 分析并调用排期工具 → 给出优化建议
  │
  └─ 无排期数据 → 发送项目描述 + 提示无排期 → AI 基于描述拆解任务 → 给出排期方案

用户在 AI 侧边栏点击预设问题
  │
  └─ 发送问题文本 → prompt-selector 匹配 system-schedule.txt → AI 调用对应工具 → 回答
```
