# 📧 邮箱通知功能测试指南

## 快速测试步骤

### 1️⃣ 配置 SMTP 服务器

#### 方案一：使用 QQ 邮箱（推荐）

1. 登录 QQ 邮箱 → 设置 → 账户
2. 开启 **POP3/SMTP 服务**
3. 生成**授权码**（不是 QQ 密码）

#### 方案二：使用 163 邮箱

1. 登录 163 邮箱 → 设置 → POP3/SMTP/IMAP
2. 开启 **SMTP 服务**
3. 生成**授权码**

#### 方案三：使用 Gmail

1. 开启 **两步验证**
2. 生成**应用专用密码**

---

### 2️⃣ 在系统中配置

访问：**设置页面** → **集成配置** → **邮件通知**

填写以下信息：

| 字段 | QQ 邮箱示例 | 163 邮箱示例 | Gmail 示例 |
|------|------------|-------------|-----------|
| SMTP 服务器 | smtp.qq.com | smtp.163.com | smtp.gmail.com |
| 端口 | 465 | 465 | 587 |
| 用户名 | your@qq.com | your@163.com | your@gmail.com |
| 密码/授权码 | 授权码 | 授权码 | 应用专用密码 |
| SSL | ✅ 开启 | ✅ 开启 | ✅ 开启 |

点击 **保存配置**

---

### 3️⃣ 发送测试邮件

1. 确保已填写正确的邮箱配置
2. 点击 **发送测试邮件** 按钮
3. 检查结果提示：
   - ✅ 成功：`测试邮件已发送至 xxx@qq.com`
   - ❌ 失败：查看错误信息

---

### 4️⃣ 检查邮件

1. 登录你的邮箱
2. 查看**收件箱**
3. 如果没有，检查**垃圾邮件**文件夹

---

## 常见问题排查

### ❌ 错误：SMTP 未配置

**原因**：SMTP 配置信息不完整

**解决**：
- 确保填写了 SMTP 服务器、端口、用户名、密码
- 点击保存后再测试

---

### ❌ 错误：连接超时

**原因**：无法连接到 SMTP 服务器

**解决**：
- 检查网络连接
- 确认 SMTP 服务器地址正确
- 确认端口号正确（465 或 587）
- 检查防火墙设置

---

### ❌ 错误：认证失败

**原因**：用户名或密码错误

**解决**：
- QQ/163 邮箱：使用**授权码**，不是登录密码
- Gmail：使用**应用专用密码**，不是账户密码
- 确保已开启 SMTP 服务

---

### ❌ 错误：SSL/TLS 错误

**原因**：加密方式不匹配

**解决**：
- 端口 465 → SSL 开启
- 端口 587 → TLS 开启（部分邮箱）

## 技术细节

### SMTP 配置存储

配置信息存储在数据库 `Setting` 表中：

- 分类：`EMAIL`
- 密码字段使用 AES-256 加密
- 支持用户级和系统级配置

### 邮件发送流程

```
1. 读取 SMTP 配置（优先用户级，其次系统级）
2. 创建 nodemailer transporter
3. 渲染 HTML 邮件模板
4. 发送邮件
5. 记录发送结果
```

### 错误处理

- 配置缺失：返回 400 错误
- 连接失败：返回 502 错误
- 认证失败：返回 502 错误
- 发送失败：返回 502 错误

### 手动测试 API

```bash

  -H "Content-Type: application/json" \
  -d '{"email":"admin@taskflow.com","password":"123456"}'

# 2. 发送测试邮件（替换 YOUR_TOKEN）
curl -X POST http://localhost:3001/api/notifications/test-email \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 数据库查看配置

```sql
-- 查看邮件配置
SELECT * FROM Setting WHERE category = 'EMAIL';

-- 查看加密的密码
SELECT key, encrypted FROM Setting 
WHERE category = 'EMAIL' AND key = 'pass';
```

## 安全建议

1. **不要提交 .env 文件**：包含 SMTP 密码
2. **使用授权码**：不要使用邮箱登录密码
3. **定期更换授权码**：建议每 3 个月更换
4. **限制发送频率**：避免被邮箱服务商封禁
