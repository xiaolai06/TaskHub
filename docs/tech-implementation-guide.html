4.4 服务层模式

// 典型 Service 函数（以 task.service.ts 为例）
export async function findAll(userId: string, filters: { page?: number; status?: string }) {
  const { page = 1, status } = filters;
  const where = {
    project: { ownerId: userId, status: { not: 'ARCHIVED' } },
    ...(status ? { status } : {}),
  };
  const [data, total] = await Promise.all([
    prisma.task.findMany({ where, skip: (page - 1) * 20, take: 20 }),
    prisma.task.count({ where }),
  ]);
  return { data, total, page };
}

5. AI 引擎实现

5.1 架构概览

基于 OpenAI SDK 的工具增强 LLM 对话系统，支持任意 OpenAI 兼容供应商（DeepSeek / OpenAI / Ollama 等）。采用 Agentic 工具循环模式：LLM 接收用户输入 → 决定调用工具 → 后端执行 → 结果返回 LLM → 循环最多 10 轮。

5.2 AIService 核心流程

1init(provider?) — 从数据库读取 AI 配置，解密 API Key，创建 OpenAI 客户端
2prompt-selector — 关键词匹配选择系统提示词（5 种角色）
3chat() — 流式请求，携带全部 40 个工具定义
4收集 text delta + tool_call delta
5如有 tool_call → 通过 toolMap 查找执行 → 结果截断 500 字符 → 追加到消息
6循环直到无 tool_call（最多 10 轮）
7SSE 事件流推送：text / tool_call / tool_result / done

8.2 统一推送函数

// backend/src/utils/push-helper.ts
export async function pushReport({ userId, title, content, type, emailSummary }) {
  // 1. 站内通知（始终）
  await notificationService.create(userId, type, title, content);
  // 2. 邮件（emailNotify=true 且有邮箱）
  if (prefs.emailNotify && user.email && emailSummary) {
    await notificationService.sendSummaryEmail(user.email, title, emailSummary);
  }
  // 3. Webhook（webhookNotify=true）
  if (prefs.webhookNotify) {
    const webhooks = JSON.parse(setting.value); // [{name, channel, url}]
    for (const wh of webhooks) {
      await notificationService.sendWebhook(wh.channel, {title, content}, wh.url);
    }
  }
}

10. 技术选型理由

🟢 为什么选 Next.js 16 + React 19
  • App Router：文件系统路由，零配置，嵌套布局天然支持仪表盘的 Sidebar + Header 结构
  • Server Components：首屏渲染不发送 JS，LCP 更快
  • Turbopack：Rust 编写的打包器，冷启动 <1s，HMR 接近即时
  • React 19：use() Hook 简化异步数据读取，ref 回调清理，更小的包体积
  • 生态成熟：shadcn/ui、React Query、Next Auth 等高质量库全部原生支持
🟢 为什么选 Express 5 + Prisma 6 + SQLite
  • Express 5：原生 async/await 错误处理，无需 express-async-errors 包装
  • Prisma 6：类型安全的 ORM，Schema 即文档，迁移工具链完整
  • SQLite：零运维成本，单文件数据库，适合一人公司场景（不需要部署 MySQL/PostgreSQL）
  • 开发效率：prisma db push 即时同步，prisma studio 可视化浏览
  • 性能足够：单用户场景下 SQLite 读写性能绰绰有余，WAL 模式支持并发读
🟢 为什么选 OpenAI SDK 做 AI 接入
  • 兼容性：OpenAI API 格式是事实标准，DeepSeek / 通义 / Ollama 等全部兼容
  • Function Calling：原生支持工具调用协议，无需手写解析逻辑
  • 流式响应：SSE 流式传输，用户实时看到 AI 输出
  • 供应商切换：只需改 baseURL 和 apiKey，代码零修改
  • 社区资源：官方维护，文档完善，TypeScript 类型完整
🟢 为什么选 Tailwind CSS 4 + shadcn/ui
  • Tailwind v4：CSS-first 配置，无需 tailwind.config.js，原生 CSS 变量支持
  • shadcn/ui：不是组件库，是组件源码——直接复制到项目中，完全可控
  • Base UI 底层：无样式原语组件，Headless 架构，无障碍访问原生支持
  • 设计一致性：统一的设计 token（色彩、间距、圆角），避免样式碎片化
  • 暗色模式：CSS 变量 + oklch 色彩空间，一套代码自动适配
🟢 为什么选 Zustand + React Query
  • 职责分离：React Query 管服务端状态（API 数据缓存），Zustand 管客户端状态（认证、UI）
  • React Query：自动重新获取、乐观更新、后台刷新、缓存失效——API 数据层的最佳实践
  • Zustand：无 Provider 包装、无样板代码、1KB 体积，比 Redux 轻 10 倍
  • 不重复存储：服务端数据不复制到客户端 store，避免数据不一致
🟢 为什么选 Zod 做校验
  • 前后端共享：同一套 Schema 定义可用于后端校验 + 前端表单 + TypeScript 类型推导
  • 声明式：链式 API 直观易读，错误信息可自定义（支持中文）
  • 性能：编译时优化，运行时校验开销极小
  • 生态：与 React Hook Form、tRPC、Express 中间件无缝集成

11. 项目创新点

11.1 🧠 AI Agent 工具调用架构

创新点：不是简单的"AI 聊天机器人"，而是一个 40 工具的 AI Agent。AI 可以直接操作数据库——创建任务、查询利润、分析客户、评估排期——所有操作通过 Function Calling 协议自动完成。

传统方案

  • 用户手动查数据 → 复制粘贴给 AI → AI 分析
  • AI 只能回答问题，不能执行操作
  • 每次对话都是独立的，没有上下文记忆

我们的方案

  • 用户说"分析一下本月利润" → AI 自动调用 5 个工具 → 直接给出结论
  • AI 可以创建任务、记录工时、推送消息（带确认机制）
  • UserMemory 长期记忆 + Conversation 上下文 = 越用越懂你

11.2 📅 智能排期引擎

创新点：不是简单的日历展示，而是一个带 AI 决策能力的排期系统。

  • 复杂度评估（assess_complexity）：根据任务描述关键词 + 技术栈自动评估复杂度，修正工时预估
  • 插单模拟（evaluate_insertion）：模拟新任务插入后对现有排期的影响，展示受影响的任务列表
  • 冲突检测（suggest_rebalance）：自动检测延期任务和工时冲突，给出严重级别和调整建议
  • 历史学习（get_historical_accuracy）：分析用户过去的工时估算准确度，学习预估偏差模式
  • 优先级排序：URGENT → HIGH → MEDIUM → LOW，同优先级按截止日期升序，每人每天 8h 上限

11.3 📊 多维度经营健康度

创新点：不是单一指标看板，而是 财务 × 客户 × 项目 × 目标四维健康度评估。

  • get_business_health：计算 4 维度评分（0-100），返回整体评级（健康/需注意/危险）
  • 6 个定时任务：晨间简报、客户雷达、利润脉搏、周报、体检、记忆沉淀——覆盖经营全场景
  • 多渠道推送：站内通知 + 邮件（HTML 模板） + 企业微信/飞书/钉钉群机器人

11.4 📨 多渠道智能推送

创新点：不是简单的通知系统，而是 AI 可驱动的多渠道推送。

  • AI 工具：AI 对话中可直接发送邮件（Markdown 自动转 HTML）或推送群消息（通过昵称查找目标）
  • 定时推送：6 个定时任务自动触发，根据用户偏好决定推送渠道
  • 昵称映射：Webhook 通过昵称（如"项目群""客户群"）标识，AI 无需记忆 URL
  • 统一推送函数：pushReport() 一行代码完成站内 + 邮件 + Webhook 三通道推送

11.5 🔒 企业级安全设计

创新点：面向个人开发者，但安全标准对标企业级。

  • AES-256-GCM 加密：API Key 存储使用认证加密，带完整性校验，防篡改
  • JWT 静默续签：剩余 <24h 自动刷新 Token，用户无感知
  • 数据隔离：所有查询强制 userId 过滤，即使同一数据库也不会泄露他人数据
  • 写操作确认：AI 所有写工具（创建/修改/删除/发送）必须用户确认后才执行
  • 限频保护：IP 滑动窗口，防止暴力登录和 API 滥用

11.6 💰 零运维一人公司方案

创新点：技术选型专为一人公司/自由职业者优化——零运维、零成本、开箱即用。

传统项目管理工具的痛点

  • Jira/Linear → 功能过剩，学习成本高
  • Notion → 缺少财务分析和 AI 能力
  • 自建系统 → 需要运维 MySQL/Redis/Nginx

智汇轻营的解决方案

  • SQLite 单文件数据库，无需安装数据库服务
  • 前后端分离但可单机部署，npm run dev 即启动
  • 金额用整数（分）存储，避免浮点精度问题
  • AI 供应商可切换（本地 Ollama = 零 API 费用）

11.7 🧩 模块化 AI 提示词系统

创新点：不是单一提示词，而是 关键词路由 + 角色切换的动态提示词系统。

  • prompt-selector：根据用户消息关键词自动选择最合适的提示词（创建/分析/排期/搜索/默认）
  • 5 种角色：每种角色有不同的工具推荐、输出格式、回复风格
  • 工具清单注入：路由层自动追加可用工具列表到提示词末尾，LLM 始终知道可用能力
  • 共享加载器：utils/prompt-loader.ts 带缓存，7 处调用共用一个实现
