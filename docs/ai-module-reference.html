二、AI 核心引擎

文件：backend/src/services/ai.service.ts

AIService 类

基于 OpenAI SDK 封装的核心引擎，负责 LLM 调用、工具循环、消息管理。

方法功能说明
init()初始化从 Setting 表读取 provider、api_key（AES 解密）、base_url、model
registerTools()注册工具将 ToolDefinition[] 转为 OpenAI function calling 格式
chat()流式对话AsyncGenerator，yield StreamEvent（text/tool_call/tool_result/done）
trimMessages()消息裁剪保留 system + 最近 8 条，控制 4000 token 以内
testConnection()连接测试发 "hi" 验证配置是否正确
StreamEvent 类型
type StreamEvent =
  | { type: 'text';        content: string }
  | { type: 'tool_call';   name: string; args: Record<string, unknown> }
  | { type: 'tool_result'; name: string; result: unknown }
  | { type: 'done' };
Function Calling 循环
1
发送消息 + system prompt + 工具定义给 LLM
2
流式读取 LLM 响应，yield text 事件给前端逐字渲染
3
检测 tool_calls：如有，执行对应 handler，yield tool_call + tool_result
4
追加结果到 messages，回到步骤 1（最多 5 轮）
5
无 tool_calls 时 yield done，对话结束
Mock 模式

当 API Key 未配置时，AIService 自动降级为 Mock 模式：从数据库查询真实业务数据，返回结构化的模拟回复，不调用 LLM。开发阶段无需配置 AI 密钥即可测试工具。

}

五、记忆系统

文件：backend/src/ai/memory-extractor.ts

双通道记忆提取

通道 1：正则匹配（实时）

  • 取最近 20 条对话消息
  • 4 个正则模式匹配关键信息
  • 存入 userMemory 表
  • 首次 confidence=0.5，重复 +0.1

通道 2：AI 提取（每周日）

  • cron job weekly-memory 触发
  • 发送对话给 LLM 提取偏好/决策
  • 输出 JSON 数组：category/key/value
  • 存入记忆库供后续对话参考
正则模式提取的记忆category
/每天工作(\d+)小时/每日工作时长习惯daily_hours
/优先级[是为]高/i优先级偏好priority_preference
/简洁|简短/i回复风格偏好reply_style
/报价|成本|花费|花钱/i成本意识cost_conscious

SSE 流式对话流程

// 前端 useAiChat.ts
const response = await fetch('/api/llm/chat/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ message, sessionId }),
  signal: abortController.signal,
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const chunk = decoder.decode(value);
  // 解析 "data: {...}" 格式的 SSE 事件
  const lines = chunk.split('\n').filter(l => l.startsWith('data: '));
