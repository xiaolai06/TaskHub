# 记账模块优化设计文档

> 基于现有代码审计，针对实用性、信息密度、视觉舒适度三个维度提出优化方案。
> 本文档仅做设计说明，不涉及实现。

---

## 一、现状问题总结

### 1.1 功能缺失（影响日常使用）

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 流水无法编辑 | 录错只能删了重录，关联项目/任务会丢失 | P0 |
| 回款无法删除/修改 | 录错金额无法修正 | P0 |
| 删除无二次确认 | 误触即丢数据，无法恢复 | P0 |
| 记账页面本身没有财务概览 | 用户需切到 Dashboard 才能看到收支情况 | P1 |
| 没有日期范围筛选 | 无法按月/季度查看特定时段的流水 | P1 |
| 没有分类筛选 | 22+ 分类只能靠搜索，效率低 | P1 |
| 操作无 Toast 反馈 | 点了按钮不知道成功还是失败 | P1 |
| 回款/订阅无筛选搜索 | 数据多了以后找不到 | P2 |
| 无导出功能 | 无法给财务/老板发报表 | P2 |

### 1.2 视觉问题（影响使用体验）

| 问题 | 表现 |
|------|------|
| 三个 Tab 信息密度不均 | 流水是紧凑列表，回款/订阅是卡片堆叠，风格割裂 |
| 顶部空间浪费 | Tab 切换下方直接是工具栏，缺少呼吸感 |
| 回款摘要卡片太扁 | 4 张卡挤在一行，数字辨识度低 |
| 订阅分类构成只显示 Top 3 | 不知道剩下的是什么 |
| 金额大小不统一 | 有的 `text-sm`，有的 `text-base`，有的 `text-xl` |
| 行内操作按钮显隐不一致 | 流水 hover 才显示删除，订阅始终显示全部操作 |

## 二、优化方案

### 2.1 流水 Tab 优化

#### A. 顶部增加「迷你财务摘要」

在 Tab 切换和工具栏之间，插入一行精简的财务快照：

```
┌─────────────────────────────────────────────────────────┐
│  本月收入 ¥12,800    本月支出 ¥8,340    结余 ¥4,460     │
│  ↑ 12% vs 上月       ↓ 5% vs 上月       ↑ 23%          │
└─────────────────────────────────────────────────────────┘
```

- 只占一行高度，不需要独立卡片
- 数字用 `font-mono text-base font-semibold`，变化率用小字 + 色块箭头
- 点击可跳转到 Dashboard 的详细财务分析
- 数据来源：已有的 `useFinanceSummary` hook，无需新接口

**设计意图**：用户打开记账页面第一眼就知道「这个月花了多少、赚了多少」，不需要切页面。

#### B. 增加日期范围筛选

在工具栏的方向筛选旁边，加一个日期范围选择器：

```
[ 全部 | 收入 | 支出 ]  [ 本月 ▾ ]  [ 来源 ▾ ]  [ 🔍 搜索... ]
```

预设选项：
- 今天
- 本周
- 本月（默认）
- 上月
- 本季度
- 本年
- 自定义范围

**设计意图**：大多数用户关心的是「这个月」的数据，而不是全部历史。默认本月 + 可切换，比默认全部更实用。

#### C. 增加分类筛选

在来源筛选旁边加分类下拉，按方向自动切换可选项：

- 全部分类
- 收入方向时：项目回款、利息、退款、补贴…
- 支出方向时：项目成本、工资、房租、餐饮…

可与方向筛选联动：选了「收入」后，分类下拉只显示收入类。

#### D. 流水行支持编辑

每行 hover 时显示编辑按钮（铅笔图标），点击打开 TransactionForm 的编辑模式：

- 复用现有 TransactionForm 组件
- 增加 `editingTx` prop，有值时为编辑模式
- 预填所有字段
- 提交调用已有的 `useUpdateTransaction` hook

#### E. 操作反馈

所有增删改操作完成后显示 Toast：
- 创建成功：`✓ 已记录 ¥xxx`
- 更新成功：`✓ 已更新`
- 删除成功：`✓ 已删除` + 3 秒内可撤销

### 2.2 回款 Tab 优化

#### A. 摘要区域重新排版

当前 4 张卡片水平排列太挤。改为 2×2 网格 + 更大的数字：

```
┌──────────────┐  ┌──────────────┐
│  应收总额     │  │  本月已收     │
│  ¥156,800    │  │  ¥42,000     │
│  3 个项目     │  │  ↑ 8%        │
└──────────────┘  └──────────────┘
┌──────────────┐  ┌──────────────┐
│  逾期未收     │  │  回款率       │
│  ¥23,500     │  │  73%         │
│  2 笔 · 15天  │  │  ▓▓▓▓▓▓░░   │
└──────────────┘  └──────────────┘
```

- 数字用 `text-2xl font-bold font-mono tabular-nums`
- 逾期卡片有红色左边框警示
- 回款率卡片内嵌微型进度条

#### B. 项目回款列表增加筛选

在列表标题栏加筛选：

```
项目回款                    [ 全部状态 ▾ ]  [ 录入回款 ]
```

筛选项：
- 全部
- 有欠款（outstanding > 0）
- 已逾期
- 已结清

#### C. 回款记录支持删除

展开项目详情后，每条回款记录右侧加删除按钮：
- 点击弹出确认框：`确认删除这笔 ¥xx 的回款记录？关联的流水记录也会一并删除。`
- 删除后同步清理关联的 Transaction

#### D. 账龄分析加百分比

当前只显示金额，补充百分比更直观：

```
0-30天   ▓▓▓▓▓▓▓▓▓▓  ¥12,000 (51%)
31-60天  ▓▓▓▓        ¥5,500  (23%)
61-90天  ▓▓▓         ¥3,000  (13%)
90+天    ▓▓          ¥3,000  (13%)
```

### 2.3 订阅 Tab 优化

#### A. 摘要卡片增加「本月已扣」

当前只有月度固定成本、年度预估、分类构成。缺少「这个月实际扣了多少」：

```
┌────────────────┐  ┌────────────────┐  ┌────────────────┐
│  月度固定成本   │  │  本月已扣       │  │  年度预估       │
│  ¥2,340/月     │  │  ¥1,890        │  │  ¥28,080       │

└────────────────┘  └────────────────┘  └────────────────┘
```

#### B. 订阅卡片增加「最近扣费记录」

展开订阅卡片后，显示最近 3 次扣费：

```
┌─ 订阅名称 ────────────────────── ¥99/月 ─┐
│  类别: 软件  下次扣费: 6月15日              │
│                                            │
│  最近扣费:                                  │

└────────────────────────────────────────────┘
```

- 数据来源：Transaction 表中 `source=SUBSCRIPTION` 且 `subscriptionId` 匹配的记录

#### C. 增加搜索和分类筛选

```
[ 🔍 搜索订阅... ]  [ 全部分类 ▾ ]  [ 添加订阅 ]
```

### 2.4 视觉优化（跨 Tab 统一）

#### A. 统一行高和间距

| 元素 | 统一规范 |
|------|----------|
| 列表行最小高度 | `min-h-[52px]` |
| 行内垂直间距 | `py-3` (12px) |
| 卡片内边距 | `p-4` (16px) |
| 区块间距 | `gap-4` (16px) |
| 摘要卡片与列表间距 | `gap-5` (20px) |

#### B. 金额显示统一

| 场景 | 样式 |
|------|------|
| 列表行内金额 | `font-mono text-sm font-medium tabular-nums` |
| 摘要卡片大数字 | `font-mono text-2xl font-bold tabular-nums` |
| 卡片内小金额 | `font-mono text-xs tabular-nums` |

#### C. 操作按钮显隐统一

所有列表行统一采用「hover 显示次要操作」模式：

- 始终显示：主操作（如展开详情）
- Hover 显示：编辑、删除
- 删除按钮用 `text-muted-foreground hover:text-red-500`，不抢视觉焦点

#### D. 空状态统一

三个 Tab 的空状态统一为：

```
     [图标 · opacity-15% · 48px]
     还没有 xxx 记录
     点击右上角按钮开始记录      ← 引导文案
```

- 图标大小统一 48px，opacity 15%
- 主文案 `text-sm text-muted-foreground`
- 引导文案 `text-xs text-muted-foreground/60`

#### E. 加载状态统一

- 统一用 `Loader2` + `animate-spin` + `size-5 text-muted-foreground`
- 居中显示，父容器 `min-h-[200px] flex items-center justify-center`

## 三、新增实用功能

### 3.1 快速记账（P1）

在工具栏右侧的「记收入」「记支出」之外，增加一个「快速记账」模式：

- 点击后弹出精简表单（只需：金额 + 类别 + 描述）
- 日期默认今天，不绑定项目/任务
- 适合随手记一笔小额支出（午餐、打车等）
- 减少打开完整 Sheet 的心理负担

**交互方式**：点击「记一笔」→ 工具栏下方展开一个轻量行内表单 → 填完点确认 → 自动收起。

### 3.2 周期统计视图（P2）

在流水 Tab 增加一个「统计」切换视图：

```
[ 列表 ] [ 统计 ]

┌─ 本月收支概览 ─────────────────────┐
│                                     │
│  收入构成（环形图）    支出构成（环形图）│
│  ┌─────────┐          ┌─────────┐   │
│  │ 项目回款  │          │ 项目成本  │   │
│  │  78%     │          │  45%     │   │
│  │ 其他     │          │ 餐饮 20% │   │
│  └─────────┘          └─────────┘   │
│                                     │
│  ┌─ 月度趋势（迷你柱状图）──────────┐│
│  │  ▓   ▓                           ││
│  │  ▓ ▓ ▓ ▓                         ││
│  │  ▓ ▓ ▓ ▓ ▓                       ││
│  │  1  2  3  4  5  6               ││
│  └──────────────────────────────────┘│
└─────────────────────────────────────┘
```

- 数据来源：已有的 `useFinanceSummary` + `useFinanceTrends`
- 图表用轻量方案：CSS 柱状图或 recharts（项目已有的图表库）
- 不需要引入新的图表库

### 3.3 批量操作（P2）

流水列表支持多选模式：

- 长按或点击「选择」按钮进入多选模式
- 左侧出现 Checkbox
- 底部浮出操作栏：`已选 3 项    [ 删除 ] [ 导出 ]`
- 批量删除有确认弹窗

### 3.4 导出功能（P2）

工具栏增加导出按钮：

```
导出 ▾
├── 导出本月流水 (CSV)
├── 导出全部流水 (CSV)
├── 导出回款报表 (CSV)
└── 导出订阅清单 (CSV)
```

- 后端新增 `GET /transactions/export?format=csv&...` 接口
- 前端用 Blob + URL.createObjectURL 触发下载
- CSV 列：日期、方向、类别、描述、金额、关联项目、关联任务、备注

## 五、防视觉疲劳策略

### 5.1 信息层次分明

```
第1层：摘要数字（大、粗、一眼看到）
第2层：工具栏（紧凑、功能明确）
第3层：列表内容（中等密度、留白适当）
第4层：辅助信息（小字、浅色、不抢注意力）
```

### 5.2 留白节奏

| 区域 | 间距 | 原因 |
|------|------|------|
| 摘要卡片之间 | `gap-3` (12px) | 紧凑，因为是同一层级信息 |
| 摘要与工具栏 | `gap-5` (20px) | 区分信息层 |
| 工具栏与列表 | `gap-3` (12px) | 工具栏和列表是配套的 |
| 日期组之间 | `py-2` + 分隔线 | 清晰分组 |
| 列表行之间 | `divide-y` (1px) | 最轻量的分隔 |

### 5.3 色彩节制

| 元素 | 色彩规则 |
|------|----------|
| 收入金额 | `text-emerald-600 dark:text-emerald-400` — 唯一的绿色 |
| 支出金额 | `text-foreground` — 不标红，减少焦虑感 |
| 逾期/警告 | `text-red-500` — 仅用于真正需要关注的情况 |
| 分类标签 | 淡色背景 + 同色系文字，饱和度 ≤ 30% |
| 次要信息 | 统一 `text-muted-foreground` |

### 5.4 减少认知负担

- **默认隐藏历史数据**：日期筛选默认「本月」，不是全部
- **折叠次要信息**：日期组默认展开今天，其他默认折叠
- **渐进式披露**：操作按钮 hover 才显示，减少视觉噪音
- **一致性**：三个 Tab 的工具栏、空状态、加载状态统一布局模式

### 7.4 Query Key 失效补全

当前部分 mutation 的 `onSuccess` 未失效所有相关 query key，需要补全：

| Mutation | 需要失效的 Key |
|----------|---------------|
| `useCreateSubscription` | 加 `['finance']` |
| `useCreatePayment` | 加 `['subscriptions', 'cost-summary']` |
| `useDeleteSubscription` | 加 `['finance']` |
| 所有 Transaction mutation | 确保包含 `['finance', 'summary']` |
