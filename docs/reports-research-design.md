# 经营看板 + 行业洞察 模块设计

## 一、经营看板（/main/reports）

### 数据来源（全部自有数据库，零爬取）
| 指标 | 数据来源 | 计算方式 |
|------|---------|---------|
| 本月收入 | Project + CostRecord | 本月完成/进行中项目的 budget 之和 |
| 本月支出 | CostRecord + Task.cost | 本月 costRecord.amount + task.cost 总和 |
| 毛利 | 收入 - 支出 | 直接减法 |
| 利润率 | 毛利 / 收入 | 百分比 |
| 项目利润排行 | Project × CostRecord × Task | profit = budget - cost，按 margin 排序 |
| 支出结构 | CostRecord group by category | 人工/外包/工具/其他占比 |
| 工时分布 | TimeEntry group by projectId | 本月工时按项目分布 |

### AI 增强功能
1. **AI 智能解读按钮** — 点击后调用 LLM 分析当前看板数据，生成文字解读
2. **异常预警** — 自动标记利润率异常低/超预算的项目
3. **AI 生成月报** — 一键生成本月经营月报（Markdown）

## 二、行业洞察（/main/research）

### 数据来源（搜索 API + AI 摘要）
1. **Tavily Search API** — 免费 1000次/月，专为 AI 设计的搜索，返回结构化结果
2. **AI 摘要** — 搜索结果 → LLM 生成中文摘要 + 相关度评分
3. **本地存储** — 搜索结果存入 SearchResult 表，收藏存入 SavedResearch 表

### AI 增强功能
1. **每日简报自动生成** — 每天 8:00 自动搜索关注话题 → AI 生成简报（已有 cron-job 框架）
2. **智能搜索** — 输入关键词 → 调搜索 API → AI 评分+中文摘要
3. **竞品追踪** — 设置关注的竞品名 → 定期搜索 → AI 总结动态
4. **机会发现** — AI 在搜索结果中标记"需求信号"

| `POST /research/save` | 收藏结果 |
| `GET /research/saved?tag=定价` | 收藏列表（标签筛选） |
| `DELETE /research/saved/:id` | 删除收藏 |

## 四、开发顺序

| 优先级 | 模块 | 后端 | 前端 |
|--------|------|------|------|
| P0 | 经营看板 - 数据聚合 | report.service.ts 4个聚合方法 | 4个统计卡片 + 项目排行 + 饼图 |
| P0 | 经营看板 - AI 解读 | POST /reports/ai-insight → LLM | "AI解读"按钮 + 结果展示 |
| P1 | 行业洞察 - 搜索+摘要 | research.service.ts + 搜索API | 搜索框 + 结果卡片 + 收藏 |

| P2 | 行业洞察 - 竞品追踪 | 定时任务扩展 | 竞品设置 + 追踪结果 |
